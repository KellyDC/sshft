## üöÄ Quick Start Guide

### 1. First Time Setup (5 min)

```yaml
# Basic upload example
- uses: kellydc/sshft@v1
  with:
    host: ${{ secrets.SSH_HOST }}
    username: ${{ secrets.SSH_USERNAME }}
    key: ${{ secrets.SSH_PRIVATE_KEY }}
    source: "dist/"
    destination: "/var/www/html/"
```

üìñ **Learn more**: [README - Uploading Files](README.md#uploading-files-to-server)

### 2. With Backup Info (5 min)

```yaml
- uses: kellydc/sshft@v1
  id: deploy
  with:
    host: ${{ secrets.SSH_HOST }}
    username: ${{ secrets.SSH_USERNAME }}
    key: ${{ secrets.SSH_PRIVATE_KEY }}
    source: "dist/"
    destination: "/var/www/html/"

- run: |
    echo "Backup: ${{ steps.deploy.outputs.backup_path }}"
    echo "Size: ${{ steps.deploy.outputs.backup_size }}"
```

üìñ **Learn more**: [EXAMPLES - Backup Usage](EXAMPLES.md#basic-backup-usage)

### 3. With Post-Script (10 min)

```yaml
- uses: kellydc/sshft@v1
  with:
    host: ${{ secrets.SSH_HOST }}
    username: ${{ secrets.SSH_USERNAME }}
    key: ${{ secrets.SSH_PRIVATE_KEY }}
    source: "dist/"
    destination: "/var/www/html/"
    post_script: |
      sudo systemctl restart nginx
      echo "Deployment complete"
```

üìñ **Learn more**: [EXAMPLES - Post-Transfer Scripts](EXAMPLES.md#post-transfer-script-execution)

### 4. With Rollback (15 min)

```yaml
- uses: kellydc/sshft@v1
  id: deploy
  with:
    # ... config
    post_script: |
      sudo systemctl restart nginx
      curl -f http://localhost/health

- name: Rollback on failure
  if: failure() && steps.deploy.outputs.backup_created == 'true'
  uses: kellydc/sshft@v1
  with:
    # ... config for rollback
```

üìñ **Learn more**: [EXAMPLES - Deploy with Rollback](EXAMPLES.md#deploy-with-rollback-capability)
